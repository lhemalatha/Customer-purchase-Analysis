    Methodology

    What this project demonstrates (for reports & viva).

    Objective

    Explore purchase patterns—category mix, time trends, regional split, and customer concentration—from transactional CSV data in the browser.

    Stack

    Semantic HTML, responsive CSS, vanilla JavaScript for parsing and aggregation, Chart.js for charts. No server-side database; data stays local unless you host the static files.

    Workflow

    Load or import CSV, map columns flexibly, filter by date/region/category/search, then export the filtered subset for Excel or further analysis.

    Expected CSV format

    Header row required; column names are matched case-insensitively. With a local server, data.csv in this folder loads automatically.

    order_id
    Unique order identifier
    customer_id
    Customer identifier
    purchase_date
    Date (YYYY-MM-DD)
    product_category
    Category label
    quantity
    Units purchased
    total_amount
    Line total (numeric)
    region
    Optional region
